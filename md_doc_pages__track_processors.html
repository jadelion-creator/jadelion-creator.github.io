Timeline Mixer Documentation
Track Processors

Track processors are executed once per track during the TimelineMixer's setup phase. They handle the setup of connections to mixers and outputs on the PlayableGraph.

Let's look at the Playable Graph generated by the SignalTrackProcessor:

Track processors manage the section highlighted in red. They connect the Timeline to any required mixer and other playables, and then to the output (which is bound to your component).

In signal tracks, this Passthrough playable is not strictly necessary but serves as a good example for creating a custom track processor with your own mixer.

Custom Track Processors

You can customize how the timeline connects to its outputs on the Playable Graph by using a custom Track Processor.

To create one, follow these steps:

  1. Create a new C# script.
  2. Add the TrackProcessor attribute, passing in the type of the timeline track.
  3. Inherit from TrackProcessor instead of MonoBehaviour.
  4. Override the ProcessTrack method and connect your playables there.

Here's an example:

[TrackProcessor(typeof(ExampleTrack))]
public class ExampleTrackProcessor : TrackProcessor
{
public override void ProcessTrack(PlayableGraph graph, TrackAsset track, ScriptPlayable<TimelinePlayable> playable, int trackIndex, Component boundComponent, ref PlayableConnectionInfo info)
{
// Create a playable output
ScriptPlayableOutput output = ScriptPlayableOutput.Create(graph, track.name);
// Optionally, create a mixer for the track
ScriptPlayable<PassthroughPlayable> passthroughMixer = ScriptPlayable<PassthroughPlayable>.Create(graph);
passthroughMixer.SetTraversalMode(PlayableTraversalMode.Passthrough);
// If you have a mixer, use the ConnectPlayable method to connect it to the timeline.
// This will store information about the connections and ensure the playback methods work as expected.
ConnectPlayable(passthroughMixer, playable, trackIndex, ref info);
output.SetSourcePlayable(passthroughMixer, 0);
output.SetUserData(boundComponent);
// If you do NOT have a mixer, or a need for one, you can just set the output's source playable to be the timeline playable without using the ConnectPlayable method.
}
}

This example would make a track processor that is applied to a custom timeline track called ExampleTrack.

‍Track processors are discovered automatically at runtime
